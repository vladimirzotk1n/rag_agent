# Агент-помощник по Трудовому Кодексу Российской Федерации

Интеллектуальная система-ассистент для поиска и объяснения статей **Трудового Кодекса РФ**.  
Проект использует **RAG-архитектуру**, собственный retriever и агентный подход для ответов на вопросы пользователей.

---

# Архитектура проекта

*(Схема будет добавлена позже)*

Основные компоненты системы:

- **Frontend** — интерфейс пользователя (раздается через Nginx)
- **FastAPI Backend** — API сервера
- **LangChain / LangGraph** — управление агентами
- **Retriever** — поиск релевантных статей
- **Qdrant** — векторное хранилище
- **Redis** — хранение истории диалогов
- **LLM** — генерация ответов и датасетов

# Подготовка данных

Данные системы представляют собой **статьи Трудового Кодекса РФ**.

## 1. Генерация датасета

В модуле `dataset_creation`:

1. Берутся статьи кодекса
2. С помощью **LLM** генерируются возможные вопросы к каждой статье
3. Формируется обучающий датасет для retriever

---

## 2. Чанкинг статей

Большие статьи разбиваются на **чанки** для улучшения поиска.

Каждый чанк содержит:

- текст статьи
- номер статьи
- метаданные

---

## 3. Индексация

Чанки индексируются в **Qdrant**:

- создаются **эмбеддинги**
- сохраняется **метаинформация**

Qdrant используется как **векторная база данных** для поиска релевантных документов.

---

# Retriever

Используется **дообученная модель E5**.

Особенности обучения:

- реализовано на **PyTorch**
- используется **InfoNCE Loss**
- применяется **in-batch negatives**
- обучение выполнено по методике из оригинальной статьи E5

Retriever используется для:

- поиска релевантных чанков
- передачи их в LLM для генерации ответа

---

# Агенты

В системе реализовано **два режима работы агента**.

## 1. Multi-hop агент

Использует:

- **LangGraph**
- **Tools**
- цепочку рассуждений

Подходит для сложных вопросов, где требуется несколько шагов поиска.

---

## 2. One-shot агент

Использует:

- **dynamic prompt**
- единичный запрос к LLM

Подходит для быстрых и простых ответов.

---

# Многопользовательская система

Система поддерживает **несколько пользователей одновременно**.

### Механизм работы

1. Для каждого нового пользователя создается `thread_id`
2. `thread_id` сохраняется в **cookies**
3. История сообщений хранится в **Redis**

Структура хранения:

```
thread_id → история сообщений пользователя
```

Это позволяет:

- сохранять контекст диалога
- изолировать пользователей друг от друга

# Переменные окружения

Перед запуском необходимо создать файл `.env`.

```bash
cp .env.example .env
```

После этого нужно указать свои ключи:

```env
OPEN_AI_KEY=your_openai_key
HF_TOKEN=your_huggingface_token
```

---

# Запуск проекта

Запуск всех сервисов выполняется через **Docker Compose**.

```bash
docker-compose up -d
```

После запуска будут подняты:

- FastAPI
- Qdrant
- Redis
- Nginx
- Frontend

---

# Завершение работы

Остановка всех сервисов:

```bash
docker-compose down
```
