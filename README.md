# ⚖️ Агент-помощник по Трудовому Кодексу РФ

Интеллектуальная система для поиска и объяснения статей Трудового Кодекса РФ.
Проект использует RAG-архитектуру, собственный Retriever и агентный подход
для ответа на вопросы пользователей.

--------------------------------------------------

🚀 Технологии

- Python (async)
- FastAPI
- LangChain / LangGraph
- PyTorch
- Qdrant
- Redis
- Docker / Docker Compose
- Nginx

--------------------------------------------------

--------------------------------------------------

📦 Подготовка данных

1. Берутся статьи Трудового Кодекса РФ
2. С помощью LLM генерируются возможные вопросы
3. Статьи разбиваются на чанки
4. Создаются эмбеддинги
5. Данные индексируются в Qdrant

--------------------------------------------------

🔎 Retriever

Используется дообученная модель E5.

Особенности обучения:
- PyTorch
- InfoNCE Loss
- in-batch negatives

Retriever ищет релевантные чанки статей,
которые затем передаются в LLM для генерации ответа.

🤖 Режимы работы агента

Переключаются в .env
```
MULTIHOP=true
```
Multi-hop агент
- LangGraph
- Tools
- цепочка рассуждений

One-shot агент
- dynamic prompt
- один запрос к LLM

--------------------------------------------------

👥 Многопользовательская система

Для каждого пользователя создается thread_id

thread_id → история сообщений

История диалога хранится в Redis,
что позволяет сохранять контекст общения.

# Как запустить:
--------------------------------------------------

⚙️ Настроить переменные окружения

Создать .env:
```
cp .env.example .env
```
Минимальные настройки:
```
OPEN_AI_KEY=your_openai_key
HF_TOKEN=your_huggingface_token
```
--------------------------------------------------

🐳 Запуск проекта
```
docker-compose up -d
```
--------------------------------------------------

🛑 Остановка проекта
```
docker-compose down
```

В России приходится использовать только с VPN.
